## 1. self-Attention计算公式

$$Attn = SoftMax(QK^{T} / \sqrt{d_{k}})V$$
``` python
class SelfAttention(nn.Module):
	super().__init__()
	def __init__(self, embed_dim):
		self.embed_dim = embed_dim
		self.Q = nn.Linear(self.embed_dim, self.embed_dim)
		self.K = nn.Linear(self.embed_dim, self.embed_dim)
		self.V = nn.Linear(self.embed_dim, self.embed_dim)
		
	def forward(self, x):
		q = self.Q(x)    # B*N*C
		k = self.K(x)    # B*N*C
		v = self.V(x)
		
		k = k.transpose(-1, -2)
		score = torch.matmul(q, k)/math.sqrt(self.embed_dim)
		attn_weight = torch.softmax(score, dim=-1)
		return torch.matmul(attn_weight, v)
```

## 2. 为什么self-Attention要除以$\sqrt{d_{k}}$


- **核心原理**：防止点积结果过大导致 Softmax 进入 **饱和区（Saturated Region）**。当维度 $d_k$ 很大时，$QK^T$ 的点积方差会变大，导致 Softmax 输出的概率分布趋向于 One-hot（极端的 0 或 1），此时梯度几乎为 0。
    
- **关键考点**：**梯度消失 (Gradient Vanishing)**。Softmax 函数在输入值很大或很小时，其导数趋近于 0。如果没有 $\sqrt{d_k}$，深层网络的权重将无法更新。

## 3. Multi-Head Attention

``` python
class SelfAttention(nn.Module):
	super().__init__()
	def __init__(self, embed_dim, num_head):
		assert embed_dim % num_head == 0
		self.embed_dim = embed_dim
		self.num_head = num_head
		self.dim = self.embed_dim // self.num_head
		
		self.Q = nn.Linear(self.embed_dim, self.embed_dim)
		self.K = nn.Linear(self.embed_dim, self.embed_dim)
		self.V = nn.Linear(self.embed_dim, self.embed_dim)
		self.proj = nn.Linear(self.embed_dim, self.embed_dim)
		
	def forward(self, x, mask=None):
		B, N, C = x.shape
		q = self.Q(x)    # B*N*C
		k = self.K(x)    # B*N*C
		v = self.V(x)
		
		# 分头
		q = q.view(B, N, self.num_head, self.dim).transpose(1, 2) #B*H*N*C
		k = k.view(B, N, self.num_head, self.dim).transpose(1, 2)
		v = v.view(B, N, self.num_head, self.dim).transpose(1, 2)
		
		# 计算attn
		k = k.transpose(-1, -2)
		score = torch.matmul(q, k)/math.sqrt(self.dim) # 除每个head的维度
		# mask
		if mask is not None:
			score = score.masked_fill(mask == 0, float('-inf'))
		attn_weight = torch.softmax(score, dim=-1)
		out = torch.matmul(attn_weight, v) # B*H*N*C
		
		out = out.transpose(1, 2).contiguous().view(B, N, C)
		out = self.proj(out)
		return out
```

## 4. Adam优化器

| SGD               | 原始的优化器              | $\theta_{t} = \theta_{t} + \alpha \cdot g_{t}$                                                                                                         |                              |
| ----------------- | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------- |
| SGD with momentum | SGD+动量              | $\theta_{t+1} = \theta_{t} + \alpha \cdot (\beta \cdot g_{t} + (1-\beta) \cdot moment_{t})$                                                            | 梯度改为动量与梯度加权                  |
| AdaGrad           | 更新学习率               | $\theta_{t+1} = \theta_{t} + \frac{\alpha}{\sqrt{\sum{g_{\tau}^{2}}}} \cdot g_{t}$                                                                     | 学习率加权，更新快的参数学习率更小，更新慢的参数学习率高 |
| RMSProp           | 类似动量方法计算权重          | $\theta_{t+1} = \theta_{t} + \frac{\alpha}{\sqrt{V_{t}}} \cdot g_{t}=\theta_{t} + \frac{\alpha}{\sqrt{\beta V_{t-1}+(1-\beta) g_{t}^{2}}} \cdot g_{t}$ | 避免二阶动量累积，导致训练过程提前结束          |
| Adam              | 融合了RMSProp和Momentum | $\theta_{t+1} = \theta_{t} + \frac{\alpha}{\sqrt{\beta V_{t-1}+(1-\beta) g_{t}^{2}}} \cdot (\beta \cdot g_{t} + (1-\beta) \cdot moment_{t})$           |                              |

### SGD (随机梯度下降)

- **优点**：计算代价极小，内存占用低；随机性有助于跳出局部最优解。
    
- **缺点**：收敛不稳定，损失函数剧烈震荡；容易陷入鞍点或峡谷区域；需要精细调节学习率。
    
### SGD with Momentum (带动量的 SGD)

- **优点**：引入惯性，有效抑制震荡；在梯度方向一致的维度加速，在波动维度相互抵消；更容易冲过平坦的鞍点。
    
- **缺点**：增加了一个动量超参数（$\gamma$）；在接近极小值时可能因为惯性“冲过头”导致越过最优解。
    

### AdaGrad (自适应梯度算法)

- **优点**：每个参数都有独立的自适应学习率；稀疏特征的参数更新更快，频繁特征的参数更新更慢；前期收敛快。
    
- **缺点**：分母上的梯度累积会不断变大，导致学习率后期迅速衰减至 0，导致模型提前停止学习。
    

### RMSProp

- **优点**：解决了 AdaGrad 学习率过早消失的问题；通过指数移动平均只保留近期梯度的信息；在非凸设置（如深度神经网络）中表现非常稳健。
    
- **缺点**：仍然需要手动设置全局初始学习率；在某些复杂场景下可能出现不收敛的摆动。
    

### Adam (自适应矩估计)

- **优点**：结合了 Momentum 的“惯性”和 RMSProp 的“自适应步长”；自带偏差修正，训练初期非常稳定；对超参数不敏感，是目前工业界的通用首选（默认方案）。
    
- **缺点**：后期可能由于学习率过小导致泛化能力不如精调后的 SGD；对某些特定任务（如某些 GAN 训练）可能不如简单优化器稳定。

## 5. 做题时对python变量作用域的一些总结

### 5.1 定义在函数中

``` python
class Solution:  
    def maxPathSum(self, root):  
        ans = float('-inf')  
  
        def dfs(node):  
            nonlocal ans  
            ...  
            ans = max(ans, val)  
  
        dfs(root)  
        return ans
```

#### ✔ 特点

- 使用 `nonlocal`
- `ans` 属于 **enclosing scope**

### 5.2 定义为实例变量

``` python
class Solution:  
    def maxPathSum(self, root):  
        self.ans = float('-inf')  
  
        def dfs(node):  
            ...  
            self.ans = max(self.ans, val)  
  
        dfs(root)  
        return self.ans
```

#### ✔ 特点

- 不用 `nonlocal`
- `ans` 属于 **对象状态（self）**

### 5.3 定义为类变量（不推荐）

``` python
class Solution:  
    ans = float('-inf')  
  
    def maxPathSum(self, root):  
        Solution.ans = float('-inf')  
  
        def dfs(node):  
            ...  
            Solution.ans = max(Solution.ans, val)  
  
        dfs(root)  
        return Solution.ans
```

#### ❌ 问题

- 多实例共享（会污染）
- 不安全

``` python
    x = 0   # 类变量

a2.x
```
