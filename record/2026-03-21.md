##  1. 从现有分支创建新分支

### ✅ 基本做法

```bash
git checkout -b new_branch
```

👉 等价于：

```bash
git branch new_branch
git checkout new_branch
```

---
### ✅ 从指定分支创建

```bash
git checkout -b new_branch origin/main
```

或：

```bash
git switch -c new_branch main
```

（推荐用 `switch`，语义更清晰）

---
### 📌 原理
- 分支本质 = 指向某个 commit 的指针
- 创建分支 = 创建一个新的指针

---
## 2. 切换分支

### ✅ 标准方式

```bash
git checkout branch_name
```

或（推荐）：

```bash
git switch branch_name
```

---
### 📌 注意点

如果有未提交修改：
- 可能会被阻止切换
- 或产生冲突

解决：
`git stash`临时保存当前工作区（未提交修改），让你可以“干净地切换分支或处理其他事情”

```bash
git stash 
git switch branch_name
git stash pop
```

##  3. 在 A 分支摘取 B 的最新 commit

---

### ✅ 操作

先拿到 B 的最新 commit：

```bash
git log B -1
```

假设 commit id 是：

```bash
abc123
```

然后在 A 分支：

```bash
git checkout A
git cherry-pick abc123
```

---

### 📌 原理

- cherry-pick = 把某个 commit **重新应用一遍**
- 会生成一个新的 commit（不是同一个）

---

### ⚠️ 注意

如果冲突：

```bash
git add .
git cherry-pick --continue
```

## 4. B 分支同步 main（无冲突）

👉 你这个场景是：

```
main: A---B---C
             \
B:            D---E
```

你想让 B 包含 main 最新内容

---

### ✅ 方法 1（推荐）：merge

```bash
git checkout B
git merge main
```

---
#### ✔ 特点

- 保留历史
- 不改变 commit
- 最安全
- 会产生一次新的提交

---

### ✅ 方法 2: rebase（更“干净”）

```bash
git checkout B
git rebase main
```

---

#### ✔ 效果

```
main: A---B---C
                  \
B:                 D'---E'
```

👉 commit 被“重写”

---
### 📌 merge vs rebase（核心区别）

| 项目                | merge | rebase |
| ----------------- | ----- | ------ |
| 历史                | 保留    | 重写     |
| 是否产生 merge commit | ✅     | ❌      |
| 是否安全              | 更安全   | 需谨慎    |
| 是否线性              | ❌     | ✅      |

---

## 5. 同步分支还有哪些方法？

除了 `merge` / `rebase`，还有一些工程上常见方式：

### ✅ 方式 1：pull（本质 = fetch + merge）

```bash
git pull origin main
```

👉 等价：

```bash
git fetch
git merge origin/main
```

---

### ✅ 方式 2：pull --rebase

```bash
git pull --rebase origin main
```

👉 等价：

```bash
git fetch
git rebase origin/main
```

---

### ✅ 方式 3：reset（危险操作 ⚠️）

```bash
git reset --hard main
```

👉 直接对齐（会丢失修改）

---

### ✅ 方式 4：cherry-pick 多个提交

```bash
git cherry-pick commit1 commit2
```

👉 精细控制同步内容

---

### ✅ 方式 5：rebase -i（交互式）

```bash
git rebase -i main
```

👉 可以：

- 合并 commit
- 删除 commit
- 修改 commit

---

### ❗ 不要在这些情况用 rebase

在“别人可能依赖这段历史”的情况下，rebase 会破坏共识，导致协作混乱

- 已经 push 到远程的公共分支
- 多人协作分支
