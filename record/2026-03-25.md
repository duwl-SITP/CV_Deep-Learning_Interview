## 1. 常见的分类损失函数

### 1.1 交叉熵损失 (Cross-Entropy Loss)

- **核心原理**：衡量模型预测概率分布与真实标签分布之间的 KL 散度。
    
- **公式**：$L = -\sum y_i \log(\hat{y}_i)$
    
- **适用场景**：最通用的多分类任务（配合 Softmax）或二分类任务（配合 Sigmoid）。
    
- **缺点**：对噪声标签敏感；在类别严重失衡时，大类样本会主导梯度更新。

### 1.2 Focal Loss

- **核心原理**：在 BCE 基础上引入调节因子 $(1-\hat{y})^\gamma$，降低易分类样本的权重，使模型集中关注**难分类样本**。
    
- **公式**：$L = -\alpha(1-\hat{y})^\gamma \log(\hat{y})$
	

其中：

- **$p_t$**：模型预测属于真实类别的概率（即 Softmax 的输出）。
    
- **$\gamma$ (Focusing Parameter)**：平滑因子，通常取 $2$。它决定了对“简单样本”的抑制程度。
    
- **$\alpha_t$ (Balancing Variant)**：平衡因子，用于平衡正负样本或各类别的比例。

- **关键考点**：解决**类别不平衡 (Class Imbalance)** 问题（如目标检测中的背景与目标）。

#### 如何解决“类别不平衡”问题？

Focal Loss 通过两个维度的“权重重构”来解决不平衡：

##### A. 解决“难易样本”不平衡（核心贡献）

在很多场景（如目标检测）中，背景（易分类样本）数量远超目标（难分类样本）。

- **机制**：引入调制因子 $(1 - p_t)^\gamma$。
    
- **逻辑**：
    
    - 如果一个样本**很容易分类**（$p_t$ 很大，如 $0.9$），那么 $(1-0.9)^2 = 0.01$。该样本对总 Loss 的贡献被削减了 100 倍。
        
    - 如果一个样本**很难分类**（$p_t$ 很小，如 $0.1$），那么 $(1-0.1)^2 = 0.81$。该样本的 Loss 几乎被完整保留。
        
- **结论**：这使得模型在训练时能够**无视那些已经学会的简单样本，死磕那些分错的困难样本**。
    
##### B. 解决“正负/类间数量”不平衡

- **机制**：引入比例因子 $\alpha_t$。
    
- **逻辑**：直接给数量少的类别（小类）分配更大的 $\alpha$ 值，给数量多的类别（大类）分配较小的 $\alpha$ 值。
    
- **结论**：这在数值层面上强行抵消了数量差异带来的梯度倾斜。

## 2. 常见的回归损失函数

### 1.1 均方误差 (Mean Squared Error, MSE / L2 Loss)

- **公式**：$L = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$
    
- **优点**：
    
    - **数学性质好**：处处可导，具有唯一的全局解（在凸函数下）。
        
    - **梯度随误差减小而减小**：当预测值接近真实值时，梯度变小，有助于模型在最优点附近稳定。
        
- **缺点**：
    
    - **对离群点极度敏感**：由于误差是平方计算的，巨大的离群点误差会被放大，导致模型为了迎合离群点而偏离正常分布。
        

### 1.2 平均绝对误差 (Mean Absolute Error, MAE / L1 Loss)

- **公式**：$L = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i|$
    
- **优点**：
    
    - **鲁棒性（Robustness）强**：对离群点不敏感。误差是线性增长的，不会像 MSE 那样被平方放大。
        
- **缺点**：
    
    - **在零点不可导**：在 $y = \hat{y}$ 处导数不连续，这可能导致模型在最优点附近产生震荡。
        
    - **梯度恒定**：无论误差多小，梯度始终不变（为 1 或 -1），不利于模型在训练后期的精细化收敛。

## 3. ReLU计算公式以及优缺点

$$f(x) = \max(0, x)$$

- 当 $x \le 0$ 时，输出为 $0$；
    
- 当 $x > 0$ 时，输出为 $x$。
    
- **导数**：
    
    $$f'(x) = \begin{cases} 0 & \text{for } x < 0 \\ 1 & \text{for } x > 0 \end{cases}$$
    
    （注：在 $x=0$ 处数学上不可导，实现时通常默认为 0 或 1）。
    
### 3.1 为什么使用 ReLU？ (用法与优点)

在 ReLU 出现之前，主流是 Sigmoid 或 Tanh。ReLU 的普及彻底改变了深层网络的训练效率：

- **缓解梯度消失 (Vanishing Gradient)**：在 $x > 0$ 区域，导数恒为 1。这意味着在反向传播时，梯度可以无损地传递到浅层，支持训练上百层的网络。
    
- **计算极快**：不涉及指数运算（Exp），计算开销极低。
    
- **稀疏激活性 (Sparsity)**：由于 $x \le 0$ 时输出为 0，这会导致网络中一部分神经元处于“关闭”状态。这种稀疏性类似于生物神经元的特性，有助于提取更鲁棒的特征并防止过拟合。
    
### 3.2 存在的问题 (Dead ReLU Problem)

**神经元“死亡”现象**

- **核心原理**：
    
    如果一个大的梯度更新导致某个神经元的权重变为负数，使得对于所有训练样本其输入 $z$ 总是 $\le 0$，那么该神经元的输出将永远是 0，且导数永远是 0。
    
- **后果**：
    
    一旦神经元“死掉”，它就再也无法在反向传播中更新权重。如果学习率（Learning Rate）设置得过大，可能会有大量神经元在训练早期集体死亡，导致网络有效容量锐减。

## 4. 手撕两层神经网络

``` python
import numpy as np

def forward(x, w, b):
    return x @ w + b

def Sigmoid(x):
    # Sigmoid 激活函数公式：1 / (1 + e^-x)
    return 1 / (1 + np.exp(-x))

B, C = 32, 64        # Batch: 32, 输入特征: 64
C1, C2 = 128, 10     # 隐藏层神经元: 128, 输出维度: 10
lr = 0.01            

x = np.random.randn(B, C)
labels = np.random.randn(B, C2)

w1 = np.random.randn(C, C1) * 0.01
b1 = np.zeros(C1)
w2 = np.random.randn(C1, C2) * 0.01
b2 = np.zeros(C2)

# ==========================
# 正向传播 (Forward Pass)
# ==========================
z1 = forward(x, w1, b1)
y1 = Sigmoid(z1)            # 隐藏层使用 Sigmoid
y2 = forward(y1, w2, b2)    # 输出层保持线性（回归任务常规做法）

# 计算 MSE 损失函数
loss = np.mean(0.5 * (y2 - labels)**2)
print(f"当前 Loss: {loss:.4f}")

# ==========================
# 反向传播 (Backward Pass)
# ==========================
dy2 = (y2 - labels) / B  
grad_w2 = y1.T @ dy2       # 注意维度
grad_b2 = np.sum(dy2, axis=0)
dy1 = dy2 @ w2.T

# Sigmoid 的导数公式：f'(x) = f(x) * (1 - f(x))
# 因为 y1 已经是 Sigmoid 的输出，所以直接用 y1 * (1 - y1)
dz1 = dy1 * (y1 * (1 - y1))

# 第一层参数梯度
grad_w1 = x.T @ dz1
grad_b1 = np.sum(dz1, axis=0)

# ==========================
# 参数更新 (Gradient Descent)
# ==========================
w1 = w1 - lr * grad_w1
b1 = b1 - lr * grad_b1
w2 = w2 - lr * grad_w2
b2 = b2 - lr * grad_b2
```
