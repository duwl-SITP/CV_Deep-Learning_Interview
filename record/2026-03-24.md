## 1.  绝对位置编码 (Sine-Cosine PE)

这是 Transformer 原型使用的固定编码，ViT 也常采用其变体。

- **公式**：
    $$PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d_{model}})$$
$$PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d_{model}})$$
    
- **要点**：$pos$ 是位置，$i$ 是维度的索引。使用正余弦是为了让模型学习相对位置，因为 $PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的线性变换。

### 1.1 为什么使用 $\sin$ 和 $\cos$？

- **核心原理：相对位置的线性可表示性**。
    
    根据三角函数的感法公式：
    
    $$\sin(\alpha + \beta) = \sin \alpha \cos \beta + \cos \alpha \sin \beta$$
    $$\cos(\alpha + \beta) = \cos \alpha \cos \beta - \sin \alpha \sin \beta$$

这意味着对于任何固定的偏移量 $k$，位置 $pos+k$ 的编码 $PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的 **线性变换**。

- **关键考点**：这使得模型能够轻而易举地学习到 **相对位置关系**。由于变换矩阵只依赖于 $k$ 而不依赖于 $pos$，模型在处理不同绝对位置但相对距离相同的 Token 时，其注意力机制逻辑是一致的。
    
- **有界性与连续性**：$\sin$ 和 $\cos$ 的值域在 $[-1, 1]$ 之间，避免了绝对数值过大导致的数值不稳定（相比于直接输入 $1, 2, 3 \dots$）。

### 1.2 为什么缩放因子是 $1/10000^{2i/d_{model}}$？

这个分母定义了波形的 **波长（Wavelength）**。

- **核心原理：多尺度特征提取**。
    
    - 当 $i$ 较小时（低维），波长很短，函数频率高，负责捕捉 **近距离、精细** 的位置关系。
        
    - 当 $i$ 较大时（高维），波长极长（可达 $2\pi \cdot 10000$），函数频率极低，负责捕捉 **长距离、粗粒度** 的位置关系。
        
-  **位置编码基数（10000）的设计动机**

	- **核心痛点：周期重叠 (Aliasing & Overlap)**
	    
	    若基数较小，$\sin$ 和 $\cos$ 函数的角频率 $\omega$ 会过大，导致波长 $\lambda$ 过短。在长序列（如 512 或 1024）中，函数会经历多个完整周期，使得不同位置 $pos$ 处的编码向量在 $d_{model}$ 维空间中产生重叠或数值过于接近，导致模型无法区分位置。
	    
	- **最终效果：全局唯一性 (Global Uniqueness)**
	    
	    这种设计确保了在极长的序列范围内，高维特征能提供单调且具有高辨识度的“底色”，从而保证每一个位置 $pos$ 生成的 $d_{model}$ 维向量在嵌入空间中都是**全局唯一且互不重叠**的几何点。

## 2. Transformer的Encoder和Decoder有哪些不同的地方？


- **核心原理**：Encoder 是**双向（Bidirectional）**的，负责提取特征并理解全局上下文；Decoder 是**单向（Auto-regressive）**的，负责根据已生成的内容预测下一个 Token。
    
- **关键考点**：Masked Multi-Head Attention 和 Cross-Attention 的存在与否。

### 2.1 注意力机制的结构 (Attention Mechanisms)

- **Encoder**：只有一层 **Self-Attention**。每个位置的 Token 都能看到序列中所有其他位置的信息（无遮掩）。
    
- **Decoder**：包含两层不同的注意力：
    
    - **Masked Self-Attention**：加入了“掩码（Mask）”，防止当前位置看到未来的信息。这保证了模型在生成第 $t$ 个词时，只能参考前 $t-1$ 个词。
        
    - **Encoder-Decoder Attention (Cross-Attention)**：Query 来自 Decoder，而 Key 和 Value 来自 Encoder 的输出。它允许 Decoder 在生成时“观察”输入序列的特征。

### 2.2 并行化能力的差异

- **Encoder**：天然支持**全并行计算**。因为所有 Token 互相可见，一次性输入即可得出所有特征。
    
- **Decoder**：
    
    - **训练阶段**：目标序列整体输入Decoder，因此需要通过 Mask 防止当前位置看到未来信息，可以实现**伪并行**（Teacher Forcing）。
        
    - **推理阶段**：必须**串行自回归**（Auto-regressive）。必须先算出第 1 个词，才能算第 2 个词。

## 3. 为什么Transformer中使用LN而不是BN？

### 3.1 变长序列的挑战 (Variable Length)

- **BN 的困境**：在 NLP 中，不同句子的长度不同。如果使用 BN，在计算 Batch 均值时，短句子后面的 Padding（填充位）会严重干扰统计量，导致计算出的均值和方差失真。
    
- **LN 的优势**：LN 的计算仅依赖于当前样本的特征。无论句子长短，它只在当前的 Token 向量内部做归一化，完全不受其他样本或 Padding 的影响。
    
### 3.2 训练与推理的一致性

- **BN 的缺点**：BN 依赖于 Batch Size。在推理（Inference）时，如果只输入一个样本，BN 必须使用训练时记录的全局均值/方差，如果测试数据分布略有偏移，效果会变差。
    
- **LN 的优势**：LN 在训练和推理时的行为完全一致，因为它不依赖于 Batch 内的其他数据。

### 3.3 实验结论

- **梯度稳定性**：在 RNN 和 Transformer 等深度序列模型中，实验证明 LN 带来的梯度稳定性远高于 BN，能有效防止深层网络中的梯度消失或爆炸。

## 4. 为什么分类任务使用交叉熵损失而不是MSE?

假定最后一层为Sigmoid函数。

### 4.1 MSE (均方误差) 的求导

- **公式**：$L = \frac{1}{2}(y - \hat{y})^2$
    
- **链式法则推导**：
    $$\frac{\partial L}{\partial w} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z} \cdot \frac{\partial z}{\partial w}$$
    
    1. $\frac{\partial L}{\partial \hat{y}} = (\hat{y} - y)$
        
    2. $\frac{\partial \hat{y}}{\partial z} = \hat{y}(1 - \hat{y})$
        
- **最终结果**：
    $$\frac{\partial L}{\partial z} = (\hat{y} - y) \cdot \sigma'(z) = (\hat{y} - y) \hat{y}(1 - \hat{y})$$
    
- **面试结论**：梯度中包含 $\hat{y}(1 - \hat{y})$。当预测值 $\hat{y}$ 接近 0 或 1 时，该项趋于 0，导致**梯度消失**，权重更新停滞。

### 4.2 Cross-Entropy的求导

- **公式**：$L = -[y \ln \hat{y} + (1-y) \ln (1-\hat{y})]$
    
- **链式法则推导**：
    $$\frac{\partial L}{\partial \hat{y}} = -(\frac{y}{\hat{y}} - \frac{1-y}{1-\hat{y}}) = \frac{\hat{y} - y}{\hat{y}(1-\hat{y})}$$
    $$\frac{\partial L}{\partial z} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z} = \frac{\hat{y} - y}{\hat{y}(1-\hat{y})} \cdot \hat{y}(1-\hat{y})$$
    
- **最终结果**：
    $$\frac{\partial L}{\partial z} = \hat{y} - y$$
    
- **面试结论**：梯度公式极其简洁，仅由**预测误差**决定。它抵消了 Sigmoid 的导数项，彻底解决了输出层饱和导致的梯度消失问题，使模型在误差大时学习极快。
